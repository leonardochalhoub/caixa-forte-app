## Summary

Caixa Forte is a pt-br personal finance app for a single user (Leonardo) that started the day with empty scaffolding and ended with: Supabase-backed multi-user auth (email+password + magic-link-less reset flow), a 4-step onboarding wizard, six account types including Conta Corrente / Renda Fixa / Renda Variável / Credit / Cash / Wallet, a categories tree with "formal income" filter, and a modernistic dashboard showing total balance with account-type breakdown panels, monthly KPIs filtered by formal-income categories, 6/12 month aggregates, 6/12 month projections, a 12-month trend chart, and upcoming scheduled transactions. Capture works in two modes: typed text (Enter submits) and a large microphone that records, live-transcribes via browser SpeechRecognition, sends audio to Groq Whisper on stop, parses via Groq llama-3.3 with confidence threshold, and auto-inserts the transaction. Every capture is logged to `capture_messages` with raw input, transcription, parse JSON, confidence, duration, and model. All tables carry RLS per `user_id`. Theme toggle (light/dark), footer with version + author links, mobile-responsive nav with active highlight, delete buttons, reconciliation dialog, and "mostrar mais" pagination round out the UX.

The original M1 scope was shipped and then some: the Groq parser and Whisper voice capture from M2 were pulled forward by explicit user request. Telegram bot (also M2) is NOT done.

## Lessons Learned

### Process

- **"Full Vision desde o dia 1" custou caro.** No brainstorm, o user escolheu Approach B (full vision) em vez de Lean Core. Isso inflou escopo imediatamente — em uma sessão, a fronteira M1/M2/M3 virou fluida: onboarding com Groq, voz, KPIs sofisticados, renda fixa/variável. Funcionou porque o user estava junto iterando em tempo real, mas acumulou dívida técnica que vamos pagar depois (ver Technical Debt).
- **Dados reais do user vieram em fragmentos.** MP 10725,88 → depois "add 6415,25 to nubank" → depois "na verdade é 22476,30 total" → depois "dividir Nubank em 3 contas". Cada correção forçou SQL manual com transferências compensadoras ocultas. Em projetos futuros: coletar snapshot completo antes de inserir qualquer dado; oferecer reconciliação via UI em vez de editar DB manualmente.
- **Auto mode acelerou muito, mas amplificou churn.** Decisões de UX (menu centralizado, mic bigger, remove label, add back label) iteraram muito rápido sem validação visual antes. Melhor: parar de 5 em 5 mudanças e validar.
- **SDD workflow funcionou bem como estrutura.** Brainstorm → Define → Design → Build → Ship deu trilhos claros. A fase Define com clarity score 15/15 + 20 few-shot examples + 32 acceptance tests é um dos artefatos mais valiosos.

### Technical

- **`@supabase/ssr` 0.5.x vs `supabase-js` 2.104 são incompatíveis.** Types inferem como `never`. Upgrade para `ssr ^0.10.0` resolveu. Checar compatibilidade de libs SDK no momento da instalação vale ouro.
- **Next 16 `typedRoutes` experimental atrapalha em iteração rápida** — rotas novas não regeneram types a tempo, `<Link>` quebra typecheck. Desabilitei. Re-habilitar só quando a set de rotas estiver estável.
- **Tailwind v4 usa @theme dentro de CSS**, não `tailwind.config.ts` como fonte da verdade. Mantive o config.ts mas as tokens de runtime (que trocam dark/light) vivem em `@theme inline` no globals.css. Funciona bem.
- **Groq structured output pt-br funciona, mas exige prompt cuidadoso.** Tendência do modelo: inventar subcategorias (ex: "pizza" → "Delivery" automático). Regras explícitas "NÃO INVENTE subcategoria mais específica do que o texto diz" corrigiram.
- **Browser SpeechRecognition + Whisper híbrido:** SpeechRecognition para live display durante gravação, Whisper para precisão na transcrição final. Só Chrome/Safari tem SR. Degradação graciosa.
- **`is_transfer` flag resolveu um problema fundamental de modelagem.** Transferências entre contas não deveriam inflar KPIs de fluxo mensal (não são income/expense reais). Flag simples + filtro em `bucketizeTransactions` + escondendo do display resolveu.
- **`is_formal_income` flag** (no lugar de "toda income conta") foi necessária quando user registrou "Saldo inicial" como income — inflou Entrada do mês. Lição: transação de setup (saldo inicial, ajuste) precisa ser marcada diferente de renda real.
- **`content-center` (align-content CSS) centraliza verticalmente em textarea** sem precisar de flex wrapper. Moderno, mas só roda em Chrome 123+/Firefox 125+/Safari 17.4+.
- **`opening_balance_cents` na tabela accounts** é cleaner que transações de "saldo inicial" porque não aparece em KPIs por default. Usamos ambos em momentos diferentes — gerou confusão.

### Communication

- **User sabe o que quer quando vê.** Especificações iniciais ("dashboard monocromático") ficaram abstratas até ele começar a usar. Toda iteração pós-deploy gerou 2-3 ajustes de UX imediatamente.
- **"Ajustar saldo" virou central.** O user quer declarar o saldo real e o app se vira pra fechar a conta — build-buildled como reconciliation dialog (ícone ⚖). Feature valiosíssima que não estava no DEFINE original.
- **Confusão com "saldo total" vs "saldo do mês".** Várias idas e voltas até separar claramente: saldo total = estado atual (openings + realized flow), saldo do mês = fluxo mensal (income - expense). Deixar isso explícito em labels do dia 1 teria economizado 4-5 iterações.

### Tools & Libraries

- **next-themes** — trivial instalar e `attribute="class"` + Tailwind dark variant = dark mode funciona em <10 linhas.
- **Groq SDK** — response_format: json_object + Zod validação = extração estrutural confiável. Grátis no tier free.
- **Recharts** — ResponsiveContainer + Line/LineChart com cores via CSS vars (`stroke="var(--color-income)"`) casa com theme dinâmico.
- **shadcn/ui** — refs + forwardRef padrão fácil de adaptar à paleta monocromática. Usado pra button/card/input/dialog/select/badge.
- **Supabase Management API** — `POST /v1/projects/{ref}/database/query` com PAT substituiu CLI inteiramente quando TLS local estava quebrado.
- **lucide-react** — ícones consistentes em tamanho; todo componente virou acessível com `aria-label`.

## Acknowledgments

This milestone was a single continuous session of live collaboration: auto-mode Claude executing, user steering in real time with fine-grained UX feedback ("make mic bigger", "text centered", "remove that label", "no, total is 22.476,30"). The rate of iteration was unusual and only possible because:
